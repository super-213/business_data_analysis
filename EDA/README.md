# 数据生成形式

# 表
## 用户表 user_unique_compare.csv：
记录用户基础画像，用于用户运营与渠道分析。

## SPU 表 spu_manages_feishu.csv：
记录商品标准单位，即不区分颜色尺码的款式级商品，用于商品管理与成本控制。

## SKU 表 sku_data_base.csv：
记录商品销售单位，即具体到颜色、尺码的可售商品，用于库存、选品与尺码运营。
销售统计表 new_sku_sales.csv：记录各 SKU 的加购、收藏、销量与退款率等行为指标，用于评估营销效果与商品健康度。

## 订单表 erp_order.csv：
记录实际订单流水，用于财务核算、履约分析与售后服务评估。

# 字段

## 用户字段包括：
用户 ID、昵称、平台、性别、所在省份。
平台采用等概率随机分配；
性别仅分两类，简洁实用，适用于大多数分析场景；
省份使用 fake.province() 确保 1000 名用户合理分散于全国 34 个省级行政区，避免城市级粒度过细导致分布失真。

## SPU 字段包括：
SPU 编号、商品标题、成本、库存、毛利率。
商品标题由基础品类与季节标签组合而成，符合电商命名惯例；
成本设定在 50 至 400 元区间，覆盖服饰类商品典型成本范围；
毛利率设定在 20% 至 60% 区间，符合行业实际

## SKU 字段包括：
SKU 编号、所属 SPU、商品名称、品类、颜色、尺码、售价。
每个 SPU 随机生成3至5个SKU，模拟实际商品多规格情形；
颜色从 7 种常见色中选取，尺码覆盖 XS 至 XL，符合国内主流尺码体系；
售价计算公式为：成本 × (1 + SPU 毛利率 + 额外溢价 5%–15%)，合理反映品牌溢价或设计附加值。

## SKU 销售统计表字段包括：
SKU 编号、销量、加购次数、收藏次数、退款率。
加购次数设定为销量的 0.8 倍至 2 倍之间，符合用户行为规律；
退款率设定在 1% 至 15% 区间，符合服饰行业平均水平。

## 订单表字段包括：
订单编号、用户 ID、SKU、数量、单价、总金额、下单日期、订单状态。
下单日期为近 90 天内随机生成，支持周度/月度趋势分析；
订单状态三种取值，可用于计算实际退款率、履约时效等指标。
